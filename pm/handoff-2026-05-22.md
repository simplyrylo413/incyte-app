## Ground Rules

1. **Never embed or re-paste images.** Describe screenshots in plain text only (e.g. "the error read: messages.270.content.1.text: cache_control cannot be set for empty text blocks").
2. **Never write instructions like "see attached screenshot" or "review the image from last time."** These can cause the next session to re-attach an image with no caption, which triggers a 400 API error that kills the entire transcript.
3. **When the user pastes an image with no caption, flag it instead of processing it.** Ask them to describe what they're seeing in text.
4. **At the start of the next session, scan this doc first and confirm it contains no image embeds or re-attach instructions before proceeding.**

## Current Context

### What we were working on

Three parallel tracks were completed or in-progress:

**1. workout-alt.html — AI ASSIST button and LOG SET fixes (DONE)**

The static cassette prototype at `src/fitlog-nextjs/public/workout-alt.html` had three bugs:
- AI ASSIST button toggled a CSS class but never actually gated the recommendation logic. Fixed by adding `state.aiOn = false` to the state object and wrapping all `showAIInsight()` calls behind `if (!state.aiOn) return`.
- The recommendation toast was referencing a `.led-ticker-text` DOM element that didn't exist. Replaced the ticker approach entirely with a fixed-position floating toast that fades in/out.
- LOG SET button was silently crashing because Supabase pre-populate restored `state.logs` entries without a `tags` array. `renderLog()` then called `log.tags.includes()` and threw. Fixed by adding `tags: s.warmup ? ['WU'] : ['WS']` in the Supabase restore block. Also added `state.userHasLogged = true` guard so Supabase async init can't overwrite sets the user has already logged (race condition fix).

**2. App-wide MPC chassis navigation (DONE)**

The workout-alt cassette nav was ported into a React component:
- `src/fitlog-nextjs/src/components/MPCNav.tsx` — four pads: INSIGHT → /momentum, TODAY → /today, PLAN → /plan, MORE → /more. Uses `usePathname()` for active state (red LED + red icon pulse), `useRouter()` for navigation.
- `src/fitlog-nextjs/src/components/MPCNav.module.css` — full MPC chassis CSS: fixed bottom, dark chassis (#161618), angled rim-light button surfaces, pin rows, LED dots, footer spec label.
- `src/fitlog-nextjs/src/components/NavGuard.tsx` updated: now imports MPCNav instead of BottomNav, suppresses nav on /login, /auth/*, and /today/workout.
- `src/fitlog-nextjs/src/app/layout.tsx` updated: paddingBottom increased from 88px to 110px to clear the taller chassis.

**3. Today page — dark theme + mobile width rebuild (DONE, not yet verified visually)**

`src/fitlog-nextjs/src/app/today/TodayPage.module.css` was systematically converted from light-mode colors to dark-mode:
- `.page` now has `max-width: 430px; margin: 0 auto; width: 100%` to constrain to mobile width.
- All `#0f1622` (ink) replaced with `#f0f4f8`, all `#5e6a82` (muted) replaced with `#8d96a5`, all `rgba(15,22,34,...)` hairlines flipped to `rgba(255,255,255,...)` equivalents.
- `.progress` panel reworked with dark glass: brand gradient at reduced opacity, dark box-shadows, dark border.
- `globals.css` body background changed from a light radial gradient to flat `#030404`.

The changes are committed and pushed to the working branch. The user has not yet visually confirmed the Today page looks correct because:
- They were checking the Netlify deployment (`incyte13.netlify.app`) which has NOT been redeployed.
- The dev server on the remote execution environment (localhost:3000) has the correct compiled output and the CSS verifies correctly when curled — dark colors and 430px constraint are confirmed in the compiled CSS.

### Key decisions made so far

- Button lighting standard: matte black with angled rim-light (Option B, locked 2026-05-20). See CLAUDE.md §14.
- MPC chassis nav replaces BottomNav app-wide. No WORKOUT tab in nav (workout-mode suppresses nav entirely via NavGuard).
- Back button in workout-alt.html routes to `/today` via `window.location.href = '/today'` (not `history.back()`).
- Dark mode is applied directly in CSS module files (hardcoded dark values), not via media query, because the app is always dark.
- Body background is always `#030404` (hardcoded in globals.css, not token-dependent).
