Set table — 3 design directions

Three reinterpretations of the table-style set tracker inside INCYTE's design language: cool blue palette, glass surfaces, mono numerics. Each option shows the same 4 sets (2 done, 1 current, 1 upcoming) so the state treatments compare directly.

A

Editorial ledger

Strict table inside a single card. Hairline rules between rows, mono numerics, columnar headers. The most data-dense option — feels like a lab sheet. Current row is marked by a left-edge accent bar and soft blue tint.

Character Densest. Best for lifters who want to scan all their sets at once. Tradeoff: more visual noise per row; column headers eat vertical space. Compatible with adding a "Δ" delta column (today vs. previous) without restructuring.
B

Pill rows, columnar

Keeps INCYTE's glass-pill row aesthetic but adopts the reference's column discipline. Header sits above as a mono eyebrow. Done rows are tinted teal; the current row has the prismatic gradient hero treatment.

Set Previous Today
01 60 × 5 60 × 5
02 100 × 3 100 × 3
03 120 × 1 120 × 1
04 137 × 5 130 × 5
Character Most consistent with the existing prototype's aesthetic. The current row borrows the hero's prismatic gradient so the "this is now" cue is unmistakable. Tradeoff: slightly less dense than A; column headers feel slightly redundant since pill structure already separates rows.
C

Typographic ledger

No card containers, no column headers. Just typography. Set number and today's value carry the visual weight; "previous" sits as a small mono subscript on the right. The least UI, most content.

03 120LB × 1R last120 × 1

Character Most editorial. Closest in spirit to the hero's typographic statement — the rows feel like content, not UI chrome. Tradeoff: lighter visual separation between rows; tap targets rely on row padding rather than a card boundary. Best paired with subtle hover/press states.
